Vamos começar pelo princípio o que é esse tal de Pelican? O Pelican é um sistema que gera sites estáticos sem necessidade de banco de dados ou backend (programação do lado do servidor). Ele funciona utilizando a linguagem Python e converte textos escritos em formatos específicos como por exemplo o Markdown, e sobre esse texto converte em arquivos .html que podem ser disponibilizados em qualquer servidor com acesso a web. O Pelican permite a instalação de plug-ins além de possuir vários temas (interface gráfica do site) a disposição gratuitamente, permitindo também que sejam criados temas personalizados a seu gosto.

Primeiro vamos a algumas colocações, aqui estou levando em consideração que você já tem o interpretador Python em uma versão acima da 3.6 instalada corretamente na sua máquina. Além disso você deve ter o Git também instalado na sua máquina.

Depois dessas colocações vamos iniciar o nosso projeto utilizando o Pelican:

pip install pelican

Quando o comando assima for executado serão instalados além do Pelican vários outros pacotes que são utlizados por ele como: jinja2, pygments, pytz, six, entre outros.

Além de instalar o próprio Pelican também devemos instalar a biblioteca Markdown para fazer com que o Pelican consigam “traduzir” o arquivo Markdown para HTML. Para instalar o Markdown usamos o comando>:

pip install markdown

Depois dos pacotes e suas dependências instalados devemos iniciar o projeto Pelican para tanto será utilizado o comando:

Pelican-quickstart

Após responder a todas as perguntas será então criada a seguinte estrutura de pastas e arquivos dentro da pasta do seu projeto:

├── content
├── output
├── pelicanconf.py
└── publishconf.py

Para que serve cada um desses arquivos e pastas, vou explicar, a pasta contente é onde serão colocados os arquivos Markdown (.md) para o Pelican interpretar e criar os arquivos HTML. A pasta output será onde ficaram armazenados os arquivos criados pelo Pelican, arquivos HTML, arquivos do tema escolhido entre outros. O arquivo pelicanconf.py é um arquivo de configuração do Pelican, vamos ver algumas de suas propriedades em breve. E o arquivo publishconf.py possui algumas configurações do Pelican para o momento da publicação do projeto. Alguns outros arquivos podem ser criados dependendo das respostas do questionário do comando pelican-quickstart.

Vamos então criar o primeiro post do nosso blog para isso vamos criar um arquivo dentro da pasta contente do nosso projeto, como estou utilizando Markdown para criação dos posts o arquivo irá se chamar hello.md, e terá o seguinte conteúdo:

Title: Hello Pelican!  
Date: 2020-03-06 12:00  
Category: Python  
Tags: pelican, markdown  
Slug: primeiro-artigo  
Author: Cleber Fonseca 
Status: published
Summary: Um resumo sobre o post do Pelican

#Testando nosso primeiro post no Pelican

Pelican é um gerador de páginas estáticas criado em python para facilitar nossa vida, na criação de blogs, landingpages ou site estáticos.

Após a criação do documento devemos então gerar os arquivos que serão fornecidos aos usuários como uma página HTML normal. Para tanto devemos usar o comando:

pelican contente -s pelicanconf.py -o output

Ao executar esse comando o que estamos fazendo é dizer ao Pelican que queremos gerar os arquivos na pasta content conforme as configurações existentes no arquivo pelicanconf.py. Assim os arquivos HTML serão criados, juntamente com os arquivos estáticos (CSS, JS, Imagens e outros) necessários para a construção da interface.

Mais informações a respeito do Pelican podem ser obtidas na documentação do mesmo.

Publicar no Github Pages

Como a ideia deste post é demonstrar como hospedar o nosso blog no Github pages vamos então instalar o pacote que fará a preparação do nosso site para subir para o github. O comando utilizado é:

pip install ghp-import

Após a conclusão da instalação poderemos rodar o comando:

ghp-import -m "Generate Pelican site" --no-jekyll -b master output

Este comando cria um "commit" para o Github que posteriormente será enviado para o servidor.

Mas antes vamos criar o repositório que receberá nossa página. Para isso você deve ter uma conta do Github ativa, acessando o Github você pode criar um novo repositório, no caso da criação do blog o mesmo de possuir o seguinte nome: "nome-do-repositorio.github.io" obrigatoriamente ele deve terminar com ".github.io". Além disso o repositório deve ser marcado como público.

Após essa criação devemos adicionar o nosso repositório ao projeto com o seguinte comando:

git remote add origin git@github.com:seu-usuario/seuusuario.github.io.git

Lembrando que neste caso estou utilizando ssh para conexão com o github, mas o comando correto aparecerá na página inicial do repositório até que seja enviado algum arquivo para o mesmo. Após a inclusão do repositório ao projeto podemos enviar o código fonte ao servidor do Github através do comando:

git push origin master

Enviando assim o nosso blog para o github. Após esse envio devemos informar ao Github que o nosso repositório é um Github page para tanto devemos acessar a aba Settings do nosso repositório e rolar a página até a parte do Github Pages e marcar qual o branch ou pasta dentro do branch estará a página, no meu caso como esse repositório é somente para o blog será selecionado o branch, master.

Assim depois de alguns segundos ao digitar no navegador: https://nome-do-repositorio.github.io veremos o nosso blog no ar.

Lembrando que sempre que for adicionado um novo post ou página teremos de rodar os comandos abaixo para publicar o blog com as alterações no ar.

pelican contente -s pelicanconf.py -o output

ghp-import -m "Generate Pelican site" --no-jekyll -b master output

git push origin master
